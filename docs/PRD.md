# Spora — Продуктовое ТЗ (PRD)

## 1. О продукте

**Spora** — бесплатное мобильное приложение для iPhone, которое напоминает пользователям о приёме грибных БАДов и содержит справочник по этим добавкам.

Приложение является дополнением к основному продукту (грибным БАДам) и не имеет собственной модели монетизации. Цель — помочь пользователю соблюдать регулярность приёма и получить максимальный эффект от продукта.

## 2. Целевая аудитория

- Покупатели грибных БАДов основного продукта.
- Возраст: 25–55 лет.
- Используют iPhone как основное устройство.
- Заинтересованы в осознанном подходе к здоровью, но забывают о регулярном приёме добавок.

## 3. Цели продукта

**Бизнес-цель:** повысить приверженность курсу приёма у пользователей основного продукта (грибных БАДов).

**Цель пользователя:** не забывать вовремя принимать БАДы и понимать, что и как принимать.

## 4. Ключевые проблемы пользователя

1. Забывает принять БАД в нужное время.
2. Не помнит, как именно принимать конкретный БАД (с едой / натощак, утром / вечером, дозировка).
3. Не уверен, что именно принимает и для чего.

## 5. MVP — состав функций

### 5.1. Напоминания о приёме

- Пользователь добавляет БАД из каталога в свой список приёма.
- Для каждого БАДа пользователь задаёт:
  - время напоминания (одно или несколько в течение дня);
  - дни недели (по умолчанию — каждый день).
- Приложение присылает push-уведомление в заданное время.
- В уведомлении видно: название БАДа и краткую подсказку (например, «принять с едой»).
- Тап по уведомлению открывает приложение на карточке этого БАДа.

### 5.2. Каталог БАДов с инструкциями

- Список всех грибных БАДов основного продукта.
- Карточка каждого БАДа содержит:
  - название (русское и латинское);
  - изображение;
  - описание / основные свойства;
  - рекомендации по приёму (дозировка, время, с едой / натощак);
  - противопоказания;
  - длительность курса (рекомендация).
- Поиск по списку — необязателен в MVP, можно отложить, если каталог небольшой (до ~15 позиций).

### 5.3. Список «мои БАДы»

- Главный экран приложения.
- Показывает все БАДы, которые пользователь добавил в свой режим приёма.
- Для каждого виден ближайший приём (время + статус).
- Можно отредактировать или удалить расписание.

### 5.4. Базовые настройки

- Включить / выключить уведомления (общий тумблер).
- Ссылка на политику конфиденциальности.

> Тихие часы в MVP не делаем — пользователи могут использовать системный режим Focus в iOS.

## 6. Что НЕ входит в MVP

Перечисленные ниже фичи отложены до будущих версий, чтобы не раздувать скоуп:

- Курсы приёма с циклами (приём → перерыв → приём).
- Дневник самочувствия / эффектов.
- Streak и статистика приёма.
- Виджеты на главном экране.
- Поддержка Apple Watch.
- Интеграция с Apple Health.
- Сканирование штрихкода / фото упаковки.
- Live Activities.
- Несколько профилей пользователей.
- Экспорт истории приёма.
- Напоминание о пополнении запаса.
- Тихие часы (используется системный Focus Mode iOS).

## 7. Пользовательские сценарии (MVP)

### Сценарий 1. Первый запуск

1. Пользователь открывает приложение.
2. Видит короткий онбординг (3–4 экрана: что делает приложение).
3. Запрос разрешения на push-уведомления.
4. Открывается каталог БАДов.

### Сценарий 2. Настройка приёма

1. В каталоге выбирает БАД.
2. Читает карточку с инструкцией.
3. Нажимает «Добавить в мой режим».
4. Выбирает время приёма (например, 09:00 и 21:00).
5. Сохраняет.
6. БАД появляется в списке «Мои БАДы».

### Сценарий 3. Получение напоминания

1. В заданное время приходит push: «Время принять Рейши. С едой».
2. Тап по push → открывается карточка БАДа.

### Сценарий 4. Отказ от БАДа

1. Заходит в «Мои БАДы».
2. Удаляет БАД из режима приёма.
3. Уведомления по нему больше не приходят.

## 8. Метрики успеха MVP

- Доля пользователей, добавивших ≥1 БАД в режим приёма (активация).
- Доля пользователей, у которых включены push-уведомления.
- Retention D7 / D30.
- Среднее число БАДов в режиме приёма у активного пользователя.

(Сбор метрик — задача послед-MVP-итерации; в первой версии достаточно зафиксировать показатели качественно через интервью с пользователями.)

## 9. Ограничения и допущения

- Приложение только для iOS, минимальная поддерживаемая версия — **iOS 17.0**.
- Только русский язык в MVP.
- Каталог БАДов хранится локально и поставляется вместе с приложением (без серверной синхронизации в MVP).
- В каталог MVP включаем 3–5 ключевых БАДов; остальные — в обновлениях.
- Никаких персональных данных, требующих регистрации, не собираем — приложение работает без аккаунта.
- Политика конфиденциальности размещается на сайте основного продукта.
